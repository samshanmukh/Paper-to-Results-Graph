## What it does

Wraps Landing.ai's **Agentic Document Extraction (ADE)** cloud API (DPT-2 model)
to turn documents into structured data. Two sub-nodes:

- **Landing.ai Parse** — document → clean Markdown + tables.
- **Landing.ai Extract** — parsed Markdown + a JSON Schema → structured fields.

Extract consumes the Markdown that Parse produces, so the usual wiring is
`Parse → Extract`.

## Landing.ai Parse

Parses PDFs, images, and spreadsheets (XLSX/CSV) into Markdown and tables.

**Lanes:**

| Lane in | Lane out | Description                |
| ------- | -------- | -------------------------- |
| `tags`  | `text`   | Extracted text as Markdown |
| `tags`  | `table`  | Extracted tables           |

| Field   | Description                                          |
| ------- | ---------------------------------------------------- |
| API Key | ADE API key (falls back to `ROCKETRIDE_LANDING_AI_KEY`)   |
| Model   | Parse model (default `dpt-2-latest`)                 |
| Region  | `US (production)` or `EU (eu-west-1)` deployment      |

## Landing.ai Extract

Pulls specific fields out of parsed Markdown using a JSON Schema you upload.
Runs **downstream of Parse** — it consumes the `text` lane.

**Lanes:**

| Lane in | Lane out    | Description                      |
| ------- | ----------- | -------------------------------- |
| `text`  | `answers`   | Extracted fields as JSON         |
| `text`  | `documents` | Extracted fields as documents    |

| Field             | Description                                                  |
| ----------------- | ----------------------------------------------------------- |
| API Key           | ADE API key                                                 |
| Extraction Schema | Uploaded `.json` JSON Schema (`format`, `x-alternativeNames`) |
| Strict            | Fail rather than partially extract when the schema mismatches |

## Pricing

ADE has a free **Explore** tier (1,000 credits to start; credits are consumed
per page). See https://landing.ai/pricing-agentic-apis.
