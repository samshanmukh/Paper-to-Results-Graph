The closed loop

app/ · 8 modules

Each step is a real module. The graph doesn't just know what papers say — it knows what has been tested.

01 · extract.py
Extract

Paper text → claims, methods, datasets, and citations as structured JSON.

02 · graph.py
Load

Idempotent MERGE into Neo4j, with CONTRADICTS edges between conflicting claims.

03 · codegen.py
Generate

Turn a method node into a runnable single-file experiment.

04 · runner.py
Run

Execute in a Daytona sandbox, with a local fallback that keeps the demo unkillable.

05 · curator.py
Curate

Write Run & Artifact nodes back with IMPLEMENTS / VALIDATES / REFUTES edges.

06 · queries.py
Evidence

Canned queries surface which claims now carry executable evidence.

07 · db_neo4j
Ask

Natural language → Cypher over the graph, answered with run ids and metrics.

08 · butterbase.py
Mirror

Every run auto-mirrored to the paper2result app — papers & run history.

Why it's different

◆
Claims become executable

A method node isn't documentation — it's a button. Generate the implementation, run it, and get a verdict backed by logs, metrics, and failure data.

⇄
Contradictions are first-class

When papers disagree, the graph shows it as a CONTRADICTS edge — then lets a run decide which side reproduces.

⚡
Sandboxed, reproducible runs

Experiments execute in an isolated Daytona sandbox. A local subprocess fallback means the loop never stalls mid-demo.

◈
Ask in plain language

A RocketRide agent translates questions to Cypher over Neo4j — "which claims now have executable evidence?" — and answers with exact numbers.

Built with

Four partner systems, one closed loop.

RocketRide

Wave agent with memory, db_neo4j (NL→Cypher) & tool_python.

Neo4j Aura

The product is the graph — papers, claims, methods, runs, edges.

Butterbase

AI gateway for every LLM call + a paper2result app storing run history.

Daytona

Sandbox backend for real execution, with a local fallback.
