Product features

Everything the system does

Not a mockup — each feature maps to a real module in app/ with verified smoke tests.

📄
extract.py · arxiv.py

Paper ingestion

Upload via arXiv URL (HTML or PDF with retry/fallback), PDF drop, or raw text. LLM extracts claims, methods, datasets, citations, and cross-paper relations into structured JSON.

◆
graph.py

Knowledge graph

Idempotent MERGE into Neo4j Aura. Papers, claims, methods, runs, and artifacts as first-class nodes. CONTRADICTS edges surface real disagreements between papers.

⚙
codegen.py

Method → code

Any method node becomes a runnable single-file experiment. Curated implementations for demo papers; live LLM codegen for newly uploaded papers via Grok.

⚡
runner.py

Sandbox execution

Daytona isolated sandboxes when DAYTONA_API_KEY is set. Local subprocess fallback keeps the demo unkillable. Captures stdout, stderr, duration, exit code.

✓
curator.py

Evidence curation

Writes Run + Artifact nodes back to Neo4j with IMPLEMENTS, VALIDATES, and REFUTES edges. Failed runs are evidence too — the graph remembers what broke.

📊
queries.py

Evidence table

Live view of which claims carry executable evidence vs. paper-only assertions. Flips in real time as runs complete — the core demo moment.

🕸
static/index.html

Interactive graph UI

vis-network visualization with color-coded node types, edge legends (CONTRADICTS / VALIDATES), zoom controls, and tabbed side panel (Evidence · Console · Ask).

◈
verigraph.pipe

Ask the research agent

RocketRide wave agent with internal memory, db_neo4j (natural language → Cypher), and tool_python. Answers cite claim ids, run ids, and exact metrics.

↺
demo_reset.py

Reset demo

One-click pristine state: clears Run/Artifact nodes, reloads papers, wipes local run dirs. Evidence returns to "no runs yet" for a clean stage demo.

☁
butterbase.py

Run history mirror

Every extraction and run auto-synced to the dedicated Butterbase paper2result app — papers and run history persisted outside Neo4j.

⇄
graph.py

Cross-paper conflicts

CONTRADICTS edges between claims from different papers. The graph doesn't hide disagreement — a run can adjudicate which side reproduces.

🧪
demo_loop.py

CLI closed loop

Headless one-command loop: python scripts/demo_loop.py wilson2017-m1 — codegen → run → curate → evidence diff, no UI required.

Why not Paper2Code?

Paper2Code stops at generation

  • Produces code stubs — no execution
  • No verdict on whether claims hold
  • Failures disappear; no graph memory
  • Contradictions between papers invisible
  • Evidence is "the model said so"

Verigraph ends in evidence

  • Code runs in Daytona or local sandbox
  • VALIDATES / REFUTES attached to claims
  • Failed runs recorded as graph citizens
  • CONTRADICTS edges + run adjudication
  • Metrics, logs, and run ids — reproducible
Architecture

The closed-loop pipeline

Eight Python modules, one RocketRide agent pipeline, four sponsor integrations — each step verified independently.

01 · extract.py

Extract

Paper text → claims, methods, datasets, citations, claim_relations as JSON. Schema-validated; fallback method if LLM returns none.

02 · graph.py

Load

Idempotent MERGE into Neo4j Aura. Label-scoped deletes on shared instance. CONTRADICTS, CITES, FROM, DESCRIBED_IN edges.

03 · codegen.py

Generate

Method node → single-file experiment. Output contract: last stdout line is JSON with metrics and claim_checks verdicts.

04 · runner.py

Run

Execute in Daytona sandbox (auto | daytona | local). Persist run record to runs/ with full capture.

05 · curator.py

Curate

Write Run + Artifact nodes. IMPLEMENTS → Method, VALIDATES/REFUTES → Claim, PRODUCED → Artifact (stdout capped 4k).

06 · queries.py

Evidence

Canned Cypher: claims, conflicts, methods, evidence. Powers the evidence table and CLI inspection.

07 · db_neo4j

Ask

RocketRide agent: NL → Cypher over the live graph. Wave memory + accuracy rule for exact metric quoting.

08 · butterbase.py

Mirror

Sync papers and runs to Butterbase paper2result app. Archive outside the graph; gateway also powers all LLM calls.

Sponsor integrations

RocketRideLocal engine on :5565. Pipeline verigraph.pipe: agent_rocketride (6 waves, internal memory) + db_neo4j (NL→Cypher) + tool_python. LLM routed through one gateway node.
Neo4j AuraCloud graph database (neo4j+s://). Product is the graph — 8 node labels, 10+ relationship types. Label-scoped ops on shared Aura instance.
ButterbaseAI gateway (OpenAI-compatible) for all LLM calls — extraction, codegen, agent. Dedicated paper2result app for papers + run history persistence.
DaytonaIsolated sandbox execution via daytona-sdk. --backend auto selects Daytona when API key present; local subprocess fallback otherwise.

LLM & models

GatewayButterbase AI gateway — ROCKETRIDE_GATEWAY_BASE_URL + ROCKETRIDE_GATEWAY_KEY
Modelx-ai/grok-4.3 (Grok) via ROCKETRIDE_GATEWAY_MODEL. Single route for extract, codegen, and agent pipeline.
Temperature0.0 for extraction/codegen (deterministic JSON). Agent uses pipeline instructions with accuracy rule for metric quoting.
Direct keyXAI_API_KEY available for direct xAI access if needed; production path uses Butterbase gateway.

Graph schema (Neo4j)

Nodes: Paper · Author · Claim · Method · Dataset · Task · Run · Artifact

Relationships:
  (Author)-[:WROTE]->(Paper)
  (Paper)-[:CITES]->(Paper)
  (Claim)-[:FROM]->(Paper)
  (Method)-[:DESCRIBED_IN]->(Paper)
  (Paper)-[:EVALUATED_ON]->(Dataset)
  (Paper)-[:ADDRESSES]->(Task)
  (Claim)-[:SUPPORTS|CONTRADICTS]->(Claim)
  (Run)-[:IMPLEMENTS]->(Method)
  (Run)-[:VALIDATES|REFUTES]->(Claim)
  (Run)-[:PRODUCED]->(Artifact)
